# The Grid v1.0.0 — Roadmap

> *"Intention is All You Need."*

This roadmap defines the path from the current prototype to a shippable v1.0.0 — the first
Cognitive Operating System that treats LLM inference as a schedulable resource and semantic
pipelines as the primary programming model.

---

## Current State (v0.1.0 — "The Prototype")

### What Already Works
- **Agent Lifecycle** — Full ProgramAgent with personality, mood, memory, relationships,
  XP, IQ, formality, spatial awareness, autonomous decision-making, delegation, and
  event-driven reactions. (~70KB of battle-tested agent logic)
- **Semantic Bus** — Broadcast channel carrying typed Events (sender, action, content)
  between all agents and the kernel.
- **AI Provider** — Dual-mode inference: local (Ollama) and cloud (OpenAI-compatible).
  AiRequest carries agent name, IQ level, position, and nearby context.
- **Persistence** — SQLite database storing agent state (personality, memory, mood, XP,
  active tasks) and inter-agent relationship affinities.
- **GridShell Parser** — Parses agent calls, pipelines (`|`), variable assignment (`=`),
  conditionals, `create_agent`, `find`, `move`, `list`, `status`, `help`.
- **SKFS Data Model** — GridFile with tags, 3D position, velocity, mass, relationships,
  semantic vectors. SpatialPhysics engine with attraction/repulsion/gravity. Tag clustering.
  Spatial queries (by tag, by radius, by semantic similarity).
- **Agent Blueprints** — TOML-serializable AgentBlueprint with personality, IQ,
  specialization, permissions, tools, and evolution config. Pre-built `think`, `implement`,
  `analyze` blueprints.
- **Terminal UI** — egui-based GUI with terminal input (`~$` prefix), event log, 3D Sector
  Map rendering, agent management commands (invoke, revoke, kill, jail, shush, reward, etc.)
- **Adversarial Arena** — Light Cycle and Melee combat modes between agents.
- **Config** — Local/cloud mode switching, API key management.

### What Doesn't Work Yet
- Semantic pipes are **parsed but don't execute** — context doesn't flow between agents.
- All agents use the **same model** — no dynamic inference scheduling.
- `.ag` blueprint files fail to load from disk (TOML schema mismatch).
- SKFS exists as data structures but isn't populated from the real filesystem.
- No permission/sandbox layer — agents touch the host FS directly.
- No memory compression or "dream cycles" — raw event logs only.
- Agent reputation doesn't influence scheduling.
- No hooks, no signals, no replay/trace.

## Milestone 1 — "First Light" (Foundation)
*Goal: Make what exists actually work end-to-end. Zero new features, just reliability.*

### 1.1 — Compilation & Stability
- [done ] Fix all remaining compilation errors across `agent.rs`, `app.rs`, `gridshell.rs`,
      `main.rs`, `filesystem.rs`
- [ ] Fix `.ag` TOML schema so blueprints load from `sys/sbin/`
- [ ] Ensure `cargo run` produces a working, interactive session
- [ ] Clean up warnings (unused variables, dead code, mut annotations)

### 1.2 — Agent Blueprint Loading
- [ ] Fix `AgentRegistry.load_from_sbin()` — validate TOML against `AgentBlueprint` struct
- [ ] Ship default blueprints: `think.ag`, `implement.ag`, `analyze.ag` in `sys/sbin/`
- [ ] `grid invoke think` spawns an agent from the `think.ag` blueprint with correct IQ,
      personality, and permissions
- [ ] `create_agent` command creates a valid `.ag` file and registers it

### 1.3 — SKFS Bootstrap
- [ ] On `grid init`, scan the working directory and populate SKFS with real files
- [ ] Assign initial tags based on file extension (`.rs` → `source,rust`, `.md` → `docs`)
- [ ] Assign initial 3D positions using deterministic hashing (current approach) but
      store them in SKFS so they persist
- [ ] `grid list` and `grid find` return real file data from SKFS

**Exit Criteria**: `cargo run` starts cleanly. Agents spawn from blueprints. SKFS reflects
the real filesystem. All grid commands work without panics.

## Milestone 2 — "Semantic Piping" (The Killer Feature)
*Goal: `think "analyze code" | implement "fix issues"` works end-to-end.*

### 2.1 — Pipeline Executor
- [ ] Implement `execute_pipeline()` in GridShell that:
  1. Spawns the first agent from its blueprint
  2. Sends the user's argument as the initial prompt
  3. Captures the agent's output (the "mental state")
  4. Injects that output as pre-prompt context into the next agent
  5. Repeats until the pipeline completes
- [ ] Pipeline output is broadcast to the Semantic Bus and displayed in the terminal
- [ ] Each stage logs: input context, agent used, model used, output context, token count

### 2.2 — Context Protocol
- [ ] Define a `PipelineContext` struct:
  ```
  { input: String, output: String, confidence: f32, agent: String, model: String, tokens: u32 }
  ```
- [ ] Agents in a pipeline receive structured context, not raw text dumps
- [ ] Agents can access `$PREV.output`, `$PREV.confidence` in their prompts

### 2.3 — Pipeline UX
- [ ] Pipeline progress indicator in the terminal (stage 1/3, stage 2/3, ...)
- [ ] `trace` command shows the context at each pipeline stage (from 2.1 logs)
- [ ] Pipeline errors show which stage failed and the partial context

**Exit Criteria**: `think "analyze this code" | implement "fix the bug"` spawns two agents
sequentially, the second receives the first's analysis as context, and the final output
appears in the terminal. `trace` shows the full pipeline state.

## Milestone 3 — "Cognitive Scheduling" (Inference as a Resource)
*Goal: The system dynamically allocates model quality based on agent priority and load.*

### 3.1 — Hybrid Inference Engine
- [ ] Refactor `ai_provider.rs` to support **simultaneous** local + cloud providers
- [ ] Define inference tiers:
  - **Tier 0 (System/Critical)**: Best available cloud model (GPT-4 class)
  - **Tier 1 (Foreground)**: Cloud model or high-quality local (Ollama large)
  - **Tier 2 (Background)**: Local model (Ollama small/medium)
  - **Tier 3 (Idle/Autonomous)**: Cheapest local model (TinyLlama class)
- [ ] `AiRequest` gains a `priority: InferenceTier` field

### 3.2 — The Scheduler
- [ ] Implement `InferenceScheduler` that:
  - Routes requests to the appropriate provider based on tier
  - Monitors system load (CPU, RAM — already measured in agent.rs)
  - Under high load: demotes all background agents one tier
  - Under critical load: pauses Tier 3 agents entirely
- [ ] User-invoked commands (via `~$`) always get Tier 0 or 1
- [ ] Autonomous agent ticks get Tier 2 or 3
- [ ] Pipeline stages get Tier 1 (elevated priority)

### 3.3 — Token Budgeting
- [ ] Track token consumption per agent per session
- [ ] `grid status` shows token budget: used/remaining per tier
- [ ] Agents that exhaust their budget get throttled (shorter max_tokens)
- [ ] `SIG_THROTTLE` event forces an agent to low-token mode
- [ ] `SIG_ABORT` cancels an in-flight inference request

**Exit Criteria**: Running `think "complex analysis"` uses GPT-4. An idle background agent
uses TinyLlama. Under CPU stress, background agents visibly downgrade. `grid status` shows
token usage per tier.

## Milestone 4 — "Living Memory" (Persistence & Identity)
*Goal: Agents remember across sessions. They dream.*

### 4.1 — Memory Compression
- [ ] On graceful shutdown, each agent's raw memory (event log) is compressed into a
      distilled summary using a cheap local model
- [ ] Summary format: key facts, relationship changes, unfinished tasks, emotional state
- [ ] On startup, agents load the summary as "long-term memory" and the last N raw events
      as "short-term memory"

### 4.2 — Dream Cycles
- [ ] When The Grid is idle for >5 minutes, trigger a "dream cycle":
  - Each agent receives its recent memory and produces a reflection
  - Reflections are stored as compressed memories
  - Personality drift: mood and formality shift slightly based on recent experiences
- [ ] `grid dreams` shows what agents dreamed about in the last cycle

### 4.3 — Agent Reputation System
- [ ] Track task completion rate, output quality (user feedback), and token efficiency
- [ ] Reputation score (0.0–1.0) per agent, persisted in SQLite
- [ ] Reputation influences inference tier: high-rep agents get promoted, low-rep demoted
- [ ] `grid reputation` shows the leaderboard
- [ ] Agents with very low reputation get flagged for "derez review"

**Exit Criteria**: Close and reopen The Grid. Agents remember yesterday's conversations in
compressed form. After idle time, agents have dreamed. `grid reputation` shows a meaningful
leaderboard.

## Milestone 5 — "The Nervous System" (Hooks, Signals, Sandboxing)
*Goal: The Grid reacts autonomously to events. Agents are safely sandboxed.*

### 5.1 — Semantic Hooks
- [ ] `hook <agent> on_write "*.rs"` — agent activates when any `.rs` file is written
- [ ] `hook <agent> on_speak <keyword>` — agent activates when keyword is mentioned
- [ ] `hook <agent> on_error` — agent activates on any error event
- [ ] Hooks stored in database, survive restarts
- [ ] `grid hooks` lists all active hooks

### 5.2 — Signal System
- [ ] `SIG_ABORT` — cancel current inference for an agent
- [ ] `SIG_THROTTLE` — force agent to low-token mode
- [ ] `SIG_PRIORITY` — temporarily elevate an agent to Tier 0
- [ ] Signals delivered via the Semantic Bus as priority events
- [ ] `grid signal <agent> <signal>` CLI command

### 5.3 — VFS Permission Layer
- [ ] Agents request file access through a permission check before touching host FS
- [ ] Default permissions per agent (from blueprint `.ag` files)
- [ ] `sandbox <agent> { allow "read_file" --scope "src/*", deny "write_file" --all }`
- [ ] Permission violations logged and broadcast as events
- [ ] `grid permissions <agent>` shows current access rights

**Exit Criteria**: A `security-auditor` agent auto-activates on `.rs` file writes.
`grid signal cargo SIG_ABORT` cancels an in-flight request. An agent without write
permission gets denied and the violation appears in the log.

## What's Deliberately Out of Scope for v1.0.0

| Feature | Why Deferred |
|---|---|
| **The Grid Protocol** (inter-Grid federation) | Requires stable local semantics first |
| **Spatial-SQL query language** | `find_by_tag` + `find_near` covers 90% of use cases |
| **Semantic embeddings** (real vector generation) | Needs embedding API integration; tag-based works for now |
| **Agent Marketplace / Package Manager** | Needs a community first |
| **Full SKFS physics simulation** | Visual polish, not core functionality |
| **Parallel pipes (`&`) and merge pipes (`>`)** | Sequential `|` is the MVP |
| **Network I/O sandboxing** | File sandboxing is the priority |

These are v1.1+ features that build on the v1.0.0 foundation.
